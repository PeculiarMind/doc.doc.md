## Overview

Implement an `ots` plugin that produces an automatic extractive summary of a document's text content using [OTS (Open Text Summarizer)](https://github.com/neopunisher/Open-Text-Summarizer). OTS is a C-based CLI tool that scores sentences by keyword frequency and returns the most significant ones as a summary.

The plugin operates as a standard pipeline plugin: it receives pre-extracted text from an upstream plugin (e.g. `markitdown` or `ocrmypdf`) via the accumulated JSON context and returns the summary as a new JSON field `summaryText`. It does not perform its own text extraction and does not write any files.

**Business Value:**
- Adds an automatic document summary to every sidecar `.md` file generated by `doc.doc.sh process`
- Enables quick at-a-glance previews of large documents in the output collection without opening the source file
- Purely stateless — no storage, no cache, no side effects; fits cleanly into the existing pipeline
- Leverages a well-established open-source tool with no runtime dependencies beyond the `ots` binary

**What this delivers:**
- `ots` plugin under `doc.doc.md/plugins/ots/` following the standard plugin structure (`descriptor.json`, `main.sh`, `install.sh`, `installed.sh`)
- `main.sh` receives accumulated pipeline JSON on stdin, extracts available text (`textContent`, `ocrText`, or `documentText` — in that priority order), pipes it to the `ots` CLI, and returns `summaryText` in the JSON output
- Configurable summary ratio (percentage of sentences to retain) via a `summaryRatio` input field, defaulting to 20%
- If no text field is available in the pipeline context, the plugin exits with code 65 (skip — ADR-004)
- `install.sh` installs OTS via the system package manager (`apt install libots-dev ots` on Debian/Ubuntu)
- `installed.sh` checks whether the `ots` binary is available on PATH

**Typical output contribution to sidecar `.md`:**
```
{{summaryText}}
```

## Acceptance Criteria

### Plugin Structure
- [ ] Plugin directory `doc.doc.md/plugins/ots/` contains `descriptor.json`, `main.sh`, `install.sh`, and `installed.sh`
- [ ] All scripts are executable (`chmod +x`)
- [ ] `descriptor.json` declares `name`, `version`, `description`, `active` flag, `dependencies`, and a `process` command with `input`/`output` JSON schemas
- [ ] `descriptor.json` passes the existing plugin descriptor validation (REQ_SEC_003)

### process Command (`main.sh`)
- [ ] Reads accumulated pipeline JSON from stdin
- [ ] Selects text input using the following priority order: `textContent` → `ocrText` → `documentText`; uses the first non-empty field found
- [ ] If none of those fields are present or all are empty, exits with code 65 (skip — ADR-004)
- [ ] Accepts an optional `summaryRatio` field (integer, 1–100, percentage of sentences to retain); defaults to `20` if absent or invalid
- [ ] Pipes the selected text to `ots --ratio <summaryRatio>` via stdin
- [ ] Returns valid JSON to stdout:
```json
{
  "summaryText": "<extracted summary sentences>",
  "summaryRatio": <effective_ratio_used>
}
```
- [ ] If `ots` exits non-zero or produces empty output, the plugin exits with code 65 (skip — ADR-004) and logs a warning to stderr
- [ ] Logs all errors to stderr; stdout contains only valid JSON or nothing
- [ ] Never writes any files to disk

### install Command (`install.sh`)
- [ ] Installs OTS via `apt install -y ots` (Debian/Ubuntu) or equivalent
- [ ] Exits 0 on success, non-zero on failure

### installed Command (`installed.sh`)
- [ ] Exits 0 if `ots` is available on PATH
- [ ] Exits non-zero if `ots` is not available

### Dependency Declaration
- [ ] `descriptor.json` declares `markitdown` and/or `ocrmypdf` as optional upstream dependencies (to signal that one of them should be active to provide text input)
- [ ] The plugin activates and runs regardless of which upstream text-extraction plugin is present; it skips gracefully (exit 65) if no text is found in the pipeline context

### Security
- [ ] Text is passed to `ots` via stdin only; no temp files, no shell interpolation of document content (REQ_SEC_005)
- [ ] `summaryRatio` input is validated as an integer in range 1–100 before use; invalid values are replaced with the default (no injection vector)
- [ ] All JSON input is validated before processing (REQ_SEC_009)

### Tests
- [ ] `tests/test_feature_0047.sh` covers:
  - Summary produced from `textContent`, `ocrText`, and `documentText` fields (priority order)
  - Skip (exit 65) when no text field is present
  - Skip (exit 65) when `ots` produces empty output
  - Custom `summaryRatio` is passed through correctly
  - Invalid `summaryRatio` falls back to default (20)
  - `installed.sh` exits correctly based on `ots` availability
  - Tests skip gracefully when `ots` is not installed
- [ ] All existing tests continue to pass

## Scope

### In Scope
- `doc.doc.md/plugins/ots/descriptor.json`
- `doc.doc.md/plugins/ots/main.sh`
- `doc.doc.md/plugins/ots/install.sh`
- `doc.doc.md/plugins/ots/installed.sh`
- Default template update: add `{{summaryText}}` placeholder to `doc.doc.md/templates/default.md`
- TDD test suite `tests/test_feature_0047.sh`

### Out of Scope
- Abstractive summarization (OTS is extractive only)
- Multi-language support beyond OTS's built-in dictionary support
- Changes to other plugins or the core processing pipeline
- Summary caching (OTS is fast enough to run on every `process` pass)

## Technical Requirements

- `ots` binary must be available on PATH at runtime
- Text is piped to `ots` via stdin using `echo "$text" | ots --ratio <n>`
- No temporary files are created
- `summaryRatio` must be validated as integer 1–100 before constructing the shell command
- JSON output assembled via `jq` (no manual string concatenation)
- Exit code 65 (ADR-004) used for all intentional skips

## Dependencies

- **ADR-004** (exit code 65 skip contract): skip behaviour when no text is available or OTS produces no output
- `markitdown` or `ocrmypdf` plugin upstream in the pipeline (optional; plugin skips gracefully without them)
