# Requirement: Default Template Provision

**ID**: req_0034

## Status
State: Accepted  
Created: 2026-02-09  
Last Updated: 2026-02-09

## Overview
The system shall provide a default Markdown template that produces comprehensive, well-formatted reports without requiring users to create custom templates.

## Description
The vision specifies `-m <markdown_template>` as a command-line parameter for template-based reporting (req_0005), but there is no requirement ensuring a usable default template is provided with the toolkit. Users should be able to generate meaningful reports immediately after installation without creating custom templates. The default template must demonstrate proper variable substitution, include all commonly extracted metadata fields, follow Markdown best practices, and serve as a reference example for users creating custom templates. An optional command-line flag should allow using the default template without specifying `-m` explicitly.

## Motivation
From the vision: "Uses the specified template (`-m`) for report formatting" and "Renders Markdown reports to the target directory (`-t`) per analyzed file and/or an aggregated report."

From req_0005: "The system shall generate Markdown reports by populating user-defined templates with extracted metadata and content insights."

While template-based reporting is defined as a requirement, the toolkit would be unusable out-of-the-box without at least one working template. Providing a high-quality default template improves usability (quality goal), reduces time-to-first-report, and serves as documentation by example for template syntax and capabilities.

## Acceptance Criteria

### Default Template File
- [ ] A default template file exists in the project repository at `scripts/template.doc.doc.md` or similar standard location
- [ ] Default template is well-documented with inline comments explaining structure and variables
- [ ] Template uses all commonly available metadata fields (filename, path, size, type, modification time, checksum)
- [ ] Template includes sections for plugin-contributed data with graceful handling of missing fields
- [ ] Template follows Markdown best practices (proper headings, formatting, structure)

### Template Content Quality
- [ ] Generated reports are human-readable and professionally formatted
- [ ] Variable substitution syntax is demonstrated for simple and complex cases
- [ ] Template includes example of conditional content (show section only if data available)
- [ ] Template includes example of iterating over plugin-provided arrays or lists
- [ ] Generated report is valid Markdown that renders correctly in standard viewers

### Command-Line Integration
- [ ] System uses default template if `-m` flag is omitted (optional `-m` flag)
- [ ] Command-line help text documents default template location and usage
- [ ] Verbose mode logs which template is being used (default vs. user-specified)
- [ ] Error message if user specifies non-existent template via `-m`

### Documentation
- [ ] Default template location documented in README
- [ ] Template customization guide explains how to create custom templates based on default
- [ ] All available variables and their data types documented
- [ ] Example custom templates provided in `examples/templates/` directory

### Extensibility
- [ ] Default template demonstrates how to access plugin-contributed data
- [ ] Template syntax supports future metadata fields without breaking compatibility
- [ ] Comments in template explain how to extend it for specific use cases
- [ ] Template serves as reference implementation for template engine capabilities

## Technical Considerations

### Default Template Example Structure
```markdown
# Analysis Report: {{filename}}

## File Information
- **Path**: {{filepath}}
- **Size**: {{filesize_human}}
- **Type**: {{mimetype}}
- **Modified**: {{modification_time}}
- **Checksum**: {{checksum_sha256}}

## Metadata
{{#if metadata}}
{{#each metadata}}
- **{{@key}}**: {{this}}
{{/each}}
{{else}}
No metadata extracted.
{{/if}}

## Plugin Analysis Results
{{#each plugins}}
### {{plugin_name}}
{{plugin_output}}
{{/each}}

---
*Generated by doc.doc.md on {{analysis_timestamp}}*
```

### Template Location Options
1. **Bundled with script**: `scripts/template.doc.doc.md` (same directory as doc.doc.sh)
2. **Standard data directory**: `/usr/share/doc.doc/template.md` (for system installs)
3. **User override**: `~/.config/doc.doc/template.md` (user customization)

### Template Engine Requirements
- Support variable substitution: `{{variable_name}}`
- Support conditionals: `{{#if condition}} ... {{/if}}`
- Support iteration: `{{#each array}} ... {{/each}}`
- Handle missing variables gracefully (empty string or configured default)
